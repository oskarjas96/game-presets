# game-presets

Game-preset definitions consumed by Vuxify. Each preset is a JSON file under
[`presets/`](presets/). A flat top-level [`index.json`](index.json) is
**generated** from those files.

## Preset format

Each preset JSON must contain, at a minimum:

| field          | type   | meaning                                         |
|----------------|--------|-------------------------------------------------|
| `display_name` | string | Human-readable name shown in the UI             |
| `version`      | string | Preset's own version string, stamped into index |
| `game_id`      | string | Stable identifier of the game; must be unique   |

`$schema_version` (preset's internal schema version) may be kept independently.

## `index.json` — generated, do not edit by hand

`index.json` is regenerated by `scripts/sync_index.py`. To update it locally:

```bash
pip install -r requirements-dev.txt   # one-time, installs blake3
python scripts/sync_index.py
```

For CI validation only:

```bash
python scripts/sync_index.py --check --diff
```

### Index entry shape

| field          | source                                                |
|----------------|-------------------------------------------------------|
| `id`           | Stable UUIDv4; persisted across re-syncs by `path`    |
| `display_name` | from preset JSON                                      |
| `version`      | from preset JSON's `version` field                    |
| `game_id`      | from preset JSON; must be unique across all presets   |
| `path`         | repo-relative posix path, e.g. `presets/foo.json`     |
| `checksum`     | blake3 hex digest of the raw preset file bytes        |

### Behavior

- Adding a preset file -> new entry, fresh `id`.
- Editing an existing preset -> its `version`, `display_name`, `game_id`, and
  `checksum` are refreshed; `id` is preserved.
- Deleting a preset file -> entry is dropped.
- Renaming a preset file -> new entry with a fresh `id` (path changed).
  This is intentional; downstream consumers can match on `path` or `game_id`.
- Two presets sharing the same `game_id` -> sync aborts with an error.

## Automation

- `.github/workflows/sync-index-pr.yml` runs on PRs and fails the check if
  `index.json` is out of sync (with a diff in the log).
- `.github/workflows/sync-index-main.yml` runs on push to `main` and
  auto-commits a regenerated `index.json` if needed.
